Introductions to GitHub and Unity

Overview

A quiz on this material can be found on this page.

The Git version control system was originally developed to manage the source code for the Linux kernel. Since then, it has become widely used for many projects, both open and closed source, on all platforms. In this course, you will submit your work on assignments by committing them into a remote GitHub repository that you set up for yourself.

Along with setting up and learning GitHub, you will also practice setting up a Unity project. GitHub and the Unity platform form the development environment for this course, and are required for every student. Unity is installed on each JB360 Lab computer, but GitHub is not. I strongly recommend saving your Unity work on an external drive (as project files can get very large very quickly) and working from a home computer to submit your work to GitHub.

GitHub Setup

Visit GitHub and create a free account, if you do not already have one. Next, download the Github client for your respective platform - Windows or Mac OSX - and sign in with your account on the client.

Click the + sign in the top left corner, and create a new repository with a name related to the course. It will ask you for a Local Path (default is fine) and a Git ignore - please select the "Unity" Git ignore. The Git ignore file is a hidden file that tracks what kind of files should never be uploaded to GitHub, files like preferences and solutions and other temporary files will be automatically excluded if you select the proper Git ignore file. Click Publish Repository in the top right corner to publish your repository online.

Next, navigate to the folder that has been created - if you used the default path, this will be Users/YOURNAME/My Documents/GitHub on Windows and in your root Users folder on Mac. Open up your folder and create a new text file named "README.md". Type something in this file and save it. The contents do not matter - this document will control what is shown underneath your repository online as a description.

The file's extension .md stands for markdown, which is a markup language that is designed to be readable in both raw and interpreted forms. You can think of it as a lightweight substitute for HTML that many programmers use. You can read about the Markdown syntax here.

Return to the GitHub client. Near the top left, you should now see a bar that says "Uncommitted Changes." Click the "Show" button next to this text, and type a brief message in the summary stating what the changes you made to the repository were. Next, click "Commit to master," and then "Sync" in the top right corner. These are the steps - Review changes, Commit, Sync - to commit any new content to your repository.

Unity

This course will cover the Unity development environment, and requires each student to have access to the development tools, either in the labs or on a home computer. The Unity tool can be downloaded at the Unity3D website.

Unity is a powerful, free-to-use development environment for game creation that allows developers to create games using its editor and program them using either a proprietary UnityScript (JavaScript-like) language, or the language C#. This course will focus on programming in C# primarily for the reason that it is a more wide-spread, useful language in future work.

Unity also has an Asset Store, from which users can download or purchase assets, including models, terrain, scripts, and all sorts of other features. However, I discourage members of this class to use the asset store if it can be avoided. I strongly encourage members of this course to work on fleshing out systems and concepts of their game, and not stress over asset creation. Several students of the art department will be here to assist in concept art and potentially in-game asset creation, but it is much better to have well-thought-out and developed game systems so that you can show the instructors and your fellow students how your game is played and why it is an enjoyable concept to further pursue developing.
