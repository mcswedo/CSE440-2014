Git

Overview

The Git version control system was originally developed to manage the source code for the Linux kernel. Since then, it has become widely used for many projects, both open and closed source. In this course, you will submit your work on assignments by committing them into a remote Git repository that I set up for you.

In order for me to create a remote Git repository for you, I need a copy of the public key on the computer your Git client program uses when connecting to remote hosts. After I receive this from you, I will create an empty remote repository for you. I will send you instructions on how to clone this repository onto your local computer. From that point on, you will work in the folder containing your local repository and push commits into your remote repository from it.

I will present these notes about Git in lecture.

Along with setting up and learning Git, you will also practice building a simple C++ program. Git and the tools needed to build distributable programs form the development environment for this course.

Instructions (Step 1)

Follow these Git submission instructions to prepare your system for using Git in this course. Make sure that you send me your public key as explained in these instructions.

Familiarize yourself with documentation on Git and perform experiments to improve your understanding and working knowledge of Git.

Instructions (Step 2)

This course will cover 3 development environments: make, Xcode and Visual Studio. Although you benefit from learning how to setup and work in all of these development environments, you only need to learn one for this course.

The following is a hello program in C++ that you should practice building and running in this assignment. 

#include <iostream>

using namespace std;

int main(int argc, char * args[]) {
    cout << "hello" << endl;
    cin.get();
}

If you are working under Linux, then you should use Gnu Make to build projects. You can also use Make to build projects under OS X and Windows.

If you are working under OS X, then you can use Xcode to build projects in this course. Xcode is a free download on the App Store.

If you are working under Windows, then you can use Visual Studio to build projects in this course. If you are working in JB 360, Visual Studio 2010 is already installed, so you don't need to install anything. On your laptop or desktop computer at home, there is a free version of Visual Studio named Visual C++ 2010 Express that you can install.

Using the tool(s) of your choice, build and run the hello program above. Make sure to include a clean target if you are using Make.

You need to wait for me to send you the clone command to create your repository before you begin step 3 of these instructions.

Instructions (Step 3)

After you receive my email with the clone command, run the command in your home directory. It will look something like the following. 

cd ~
git clone your_name@web1.ias.csusb.edu:repo.git 441

This will create a folder named 441 in your home directory, so make sure a folder by this name does not exist prior to running the command. From this point on, I will refer to this folder by the expression ~/441.

Tilda ~ represents a home directory on Unix-like operating systems such as Linux and OS X.

The git tutorials and git book talk about the git init command, and you may have experimented with it. However, do not use git init to create the ~/441 folder. Instead, rely on the git clone command to create this folder for you. In our case, git clone is a simpler alternative to the git init command.

In ~/441 you will see a file named README.md, which contains some instructions to help you configure git on your system. Read and follow these instructions.

The file's extension .md stands for markdown, which is a markup language that's designed to be readable in both raw and interpreted forms. You can think of it as a lightweight substitute for HTML that many programmers use. You should read about the Markdown syntax.

For this assignment, create a folder named git within ~/441. In this folder, you will practice setting up a project designed to build a shared code base to many targets, including OS X, Windows, iOS and Android.

Under ~/441/git create a folder named src and place the source code for the hello program there. Place other project files in a separate folder. For example, if you create a Visual Studio project to build the hello program, then you could put your project files under ~/441/git/win. You can use your own naming convention, but keep platform specific project files separate from the platform independent source code. If you are using make, then you can put the Makefile under ~/441/git.

Add patterns to ~/441/.gitignore to exclude derived, temporary and user-specific files from the Git repository such as a.out.

I will deduct points from your score on this assignment if you push derived, temporary or user-specific files into your remote repository. Use git status to see examine the state of your working directory and index before committing.
